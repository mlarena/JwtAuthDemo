ЗАДАНИЕ НА РАЗРАБОТКУ ПРИЛОЖЕНИЯ
Название проекта: JwtAuthDemo 

Тип приложения: ASP.NET Core MVC с JWT-аутентификацией и авторизацией

База данных: PostgreSQL

Цель: Разработать полнофункциональную систему аутентификации и авторизации с разграничением прав доступа на основе ролей и разрешений (RBAC + PBAC).

🛡️ СИСТЕМА РОЛЕЙ И ПРАВ
Стандартные роли:
Роль	Описание	Права
SuperAdmin	Полный доступ	*:* (все ресурсы, все действия)
Admin	Управление системой	users:*, roles:*, audit:read
Manager	Управление контентом	users:read, users:update, reports:*
User	Обычный пользователь	profile:*, dashboard:read
Guest	Гостевой доступ	dashboard:readonly
Примеры Permission:
text
users:read       - просмотр пользователей
users:create     - создание пользователей
users:update     - редактирование пользователей
users:delete     - удаление пользователей
users:lock       - блокировка пользователей
roles:read       - просмотр ролей
roles:create     - создание ролей
roles:update     - редактирование ролей
roles:delete     - удаление ролей
permissions:manage - управление правами
audit:read       - просмотр логов
profile:edit     - редактирование профиля
dashboard:read   - просмотр дашборда


Вход в систему:
text
Пользователь → Login (POST)
    ↓
Валидация учетных данных
    ↓
Проверка: активен ли пользователь, не заблокирован
    ↓
Генерация Access Token + Refresh Token
    ↓
Сохранение Refresh Token в БД
    ↓
Установка JWT в HttpOnly Cookie
    ↓
Перенаправление на Dashboard
    ↓
Запись в AuditLog: "Login Success"
2. Авторизация (Authorization):
text
Запрос к защищенному ресурсу
    ↓
Middleware проверяет JWT из Cookie
    ↓
Валидация токена (подпись, срок действия)
    ↓
Извлечение claims (UserId, Roles, Permissions)
    ↓
Проверка прав через Policy/Claims
    ↓
Доступ разрешен/запрещен
3. Обновление токена:
text
JWT истек → Запрос к /refresh-token
    ↓
Проверка Refresh Token из Cookie
    ↓
Поиск в БД (не истек, не отозван)
    ↓
Генерация новой пары токенов
    ↓
Обновление Refresh Token в БД
    ↓
Отправка новых токенов в Cookie
4. Выход:
text
Пользователь → Logout (POST)
    ↓
Удаление Refresh Token из БД
    ↓
Очистка Cookie
    ↓
Перенаправление на Login
    ↓
Запись в AuditLog: "Logout"

🛡️ НАСТРОЙКА JWT И АВТОРИЗАЦИИ
Конфигурация JWT:
csharp
// appsettings.json
{
  "JwtSettings": {
    "SecretKey": "ваш-секретный-ключ-минимум-256-бит",
    "Issuer": "SecureGateway",
    "Audience": "SecureGatewayUsers",
    "AccessTokenExpirationMinutes": 15,
    "RefreshTokenExpirationDays": 7
  }
}
Настройка авторизации:
Атрибуты: [Authorize], [AllowAnonymous]

Policy-based авторизация:

[Authorize(Policy = "CanManageUsers")]

[Authorize(Roles = "Admin")]

Resource-based авторизация: Проверка прав на конкретный объект

Custom Authorization Filter: Для проверки прав доступа
