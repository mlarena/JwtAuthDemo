# Пользовательская инструкция для администратора: JwtAuthDemo

## 1. Введение

JwtAuthDemo — система управления пользователями, ролями и разрешениями с JWT-аутентификацией. Данная инструкция описывает работу администратора в системе.

---

## 2. Первичный вход в систему

### Учетные данные по умолчанию

| Параметр | Значение |
|----------|----------|
| URL | http://localhost:5044 |
| Логин | `superadmin` |
| Email | `superadmin@admin.com` |
| Пароль | `Ps$$word01!` |

### Процедура входа

1. Перейдите по адресу http://localhost:5044
2. Система перенаправит на страницу входа
3. Введите логин или Email: `superadmin`
4. Введите пароль: `Ps$$word01!`
5. Нажмите **«Войти»**
6. Вы будете перенаправлены на Dashboard

---

## 3. Dashboard (Главная панель)

После входа отображается панель управления со следующими젯ами:

- **Всего пользователей** — общее количество зарегистрированных пользователей
- **Активных сегодня** — количество входов за сегодня
- **Новых за неделю** — количество новых регистраций за последние 7 дней
- **Онлайн сейчас** — количество пользователей с активными сессиями
- **Последние действия** — лента последних действий из журнала аудита
- **Распределение по ролям** — диаграмма распределения пользователей по ролям

---

## 4. Управление пользователями

### 4.1. Просмотр списка пользователей

1. В меню выберите **«Пользователи»**
2. Отображается таблица со списком всех пользователей
3. **Столбцы:** Имя, Email, Роли, Статус, Последний вход, Действия

### 4.2. Поиск и фильтрация

Используйте форму поиска над таблицей:
- **Поиск** — по имени, email, имени/фамилии
- **Роль** — фильтр по роли (SuperAdmin, Admin, Manager, User, Guest)
- **Статус** — Активные / Неактивные / Заблокированные

### 4.3. Создание пользователя

1. Нажмите кнопку **«Создать»**
2. Заполните форму:
   - Имя пользователя (обязательно)
   - Email (обязательно)
   - Имя, Фамилия (необязательно)
   - Пароль (минимум 8 символов, заглавная + строчная + цифра + спецсимвол)
   - Подтверждение пароля
3. Выберите роли (отметьте галочками)
4. Нажмите **«Создать»**

### 4.4. Просмотр профиля пользователя

- Нажмите **иконку «Глаз»** в столбце «Действия»
- Отображается полная информация о пользователе:
  - Личные данные
  - Роли
  - Статус
  - История входов
  - Активные сессии

### 4.5. Назначение ролей

1. Нажмите **иконку «Роль»** в столбце «Действия»
2. Отметьте нужные роли галочками
3. Нажмите **«Сохранить»**

### 4.6. Блокировка / Разблокировка

- Нажмите **иконку «Замок»** для блокировки/разблокировки
- Заблокированный пользователь не сможет войти в систему

### 4.7. Удаление пользователя

- Нажмите **иконку «Корзина»**
- Подтвердите удаление в диалоговом окне
- Пользователь будет удален из системы

---

## 5. Управление ролями

### 5.1. Просмотр списка ролей

1. В меню выберите **«Роли»**
2. Отображается таблица с ролями:
   - Название
   - Описание
   - Системная (да/нет)
   - Количество пользователей
   - Количество разрешений

### 5.2. Создание роли

1. Нажмите **«Создать»**
2. Введите название и описание
3. Нажмите **«Создать»**

### 5.3. Редактирование роли

- Нажмите **иконку «Карандаш»** (только для несистемных ролей)

### 5.4. Управление разрешениями

1. Нажмите **иконку «Щит»** для роли
2. Отображается список разрешений, сгруппированных по ресурсам:
   - **users** — управление пользователями
   - **roles** — управление ролями
   - **permissions** — управление разрешениями
   - **audit** — журнал аудита
   - **profile** — профиль пользователя
   - **dashboard** — дашборд
3. Отметьте нужные разрешения галочками
4. Нажмите **«Сохранить разрешения»**

### 5.5. Удаление роли

- Нажмите **иконку «Корзина»** (только для несистемных ролей без пользователей)

---

## 6. Журнал аудита

### 6.1. Просмотр журнала

1. В меню выберите **«Аудит»**
2. Отображается таблица со всеми действиями в системе:
   - ID
   - Пользователь
   - Действие
   - Ресурс
   - IP-адрес
   - Статус (Успех/Ошибка)
   - Дата и время

### 6.2. Фильтрация

- **Действие** — поиск по типу действия
- **Статус** — Успех / Ошибка
- **Период** — дата «С» и «По»

### 6.3. Типы записей

| Действие | Описание |
|----------|----------|
| Login Success | Успешный вход |
| Login Failed | Неудачная попытка входа |
| Logout | Выход из системы |
| User Registered | Регистрация пользователя |
| Email Confirmed | Подтверждение email |
| Password Reset Requested | Запрос сброса пароля |
| Password Reset Completed | Сброс пароля выполнен |
| Profile Updated | Обновление профиля |
| User Created | Создание пользователя |
| User Deleted | Удаление пользователя |
| User Lock Toggled | Блокировка/разблокировка |
| Role Created / Updated / Deleted | Управление ролями |
| Roles Assigned | Назначение ролей |
| Permissions Updated | Обновление разрешений |

---

## 7. Профиль пользователя

### 7.1. Просмотр профиля

1. Нажмите на имя пользователя в верхнем меню
2. Выберите **«Профиль»**
3. Отображается:
   - Личная информация
   - Роли
   - Статус
   - История входов
   - Активные сессии

### 7.2. Редактирование профиля

1. Нажмите **«Редактировать»**
2. Измените данные (FirstName, LastName, Email)
3. Нажмите **«Сохранить»**

---

## 8. Стандартные роли

| Роль | Описание | Права |
|------|----------|-------|
| SuperAdmin | Полный доступ | Все разрешения |
| Admin | Управление системой | users:*, roles:*, audit:read |
| Manager | Управление контентом | users:read, users:update, dashboard:read |
| User | Обычный пользователь | profile:edit, dashboard:read |
| Guest | Гостевой доступ | dashboard:read |

---

## 9. Выход из системы

1. Нажмите на имя пользователя в верхнем меню
2. Выберите **«Выйти»**
3. Вы будете перенаправлены на страницу входа
4. Refresh Token будет удален из БД
5. JWT Cookie будет удален

---

## 10. Рекомендации по безопасности

1. **Смените пароль** суперадминистратора после первого входа
2. **Не используйте** SuperAdmin для повседневной работы — создайте отдельного Admin
3. **Регулярно просматривайте** журнал аудита на подозрительную активность
4. **Не передавайте** учетные данные третьим лицам
5. **Используйте HTTPS** в production-среде
