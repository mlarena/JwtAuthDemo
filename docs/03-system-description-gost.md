# ОПИСАНИЕ СИСТЕМЫ

## JwtAuthDemo

### Система аутентификации и авторизации

---

## 1. ОБЩИЕ СВЕДЕНИЯ

### 1.1. Наименование и назначение системы

**Наименование:** JwtAuthDemo

**Назначение:** Система предназначена для аутентификации и авторизации пользователей в веб-приложении с использованием JSON Web Token (JWT) и модели разграничения прав доступа на основе ролей и разрешений (RBAC + PBAC).

### 1.2. Область применения

Система применяется для:
- управления учётными записями пользователей;
- аутентификации пользователей в веб-приложении;
- разграничения доступа к функциональным возможностям системы;
- ведения журнала аудита действий пользователей.

### 1.3. Нормативные документы

- ГОСТ 34.003-90 «Информационная технология. Комплекс стандартов на автоматизированные системы. Термины и определения»
- ГОСТ 19.001-80 «Система стандартов по автоматизированным системам. Общие положения»
- ГОСТ Р 50922-2006 «Информационная технология. Защита информации. Основные термины и определения»
- RFC 7519 «JSON Web Token (JWT)»
- RFC 6749 «The OAuth 2.0 Authorization Framework»

## 2. ХАРАКТЕРИСТИКА СИСТЕМЫ

### 2.1. Функциональные характеристики

Система реализует следующие функциональные модули:

#### 2.1.1. Модуль аутентификации (Authentication)

| Код operation | Наименование operation | Описание |
|---------------|----------------------|----------|
| AUTH-01 | Регистрация пользователя | Создание новой учётной записи с валидацией данных |
| AUTH-02 | Вход в систему | Проверка учётных данных, генерация JWT-токена |
| AUTH-03 | Выход из системы | Удаление токена, очистка сессии |
| AUTH-04 | Обновление токена | Продление JWT-токена с помощью refresh-токена |
| AUTH-05 | Подтверждение email | Верификация адреса электронной почты |
| AUTH-06 | Сброс пароля | Восстановление доступа через email |

#### 2.1.2. Модуль управления пользователями (User Management)

| Код operation | Наименование operation | Описание |
|---------------|----------------------|----------|
| USR-01 | Просмотр списка пользователей | Табличный вывод с пагинацией и фильтрацией |
| USR-02 | Просмотр профиля | Отображение个人信息 и истории входов |
| USR-03 | Редактирование профиля | Изменение имени, фамилии, email |
| USR-04 | Создание пользователя | Ручное создание учётной записи администратором |
| USR-05 | Удаление пользователя | Удаление учётной записи из системы |
| USR-06 | Блокировка/разблокировка | Временная блокировка доступа |
| USR-07 | Назначение ролей | Изменение ролевой принадлежности пользователя |

#### 2.1.3. Модуль управления ролями (Role Management)

| Код operation | Наименование operation | Описание |
|---------------|----------------------|----------|
| ROL-01 | Просмотр списка ролей | Отображение ролей с количеством пользователей |
| ROL-02 | Создание роли | Добавление новой роли в систему |
| ROL-03 | Редактирование роли | Изменение параметров роли |
| ROL-04 | Удаление роли | Удаление несистемной роли |
| ROL-05 | Управление разрешениями | Назначение разрешений роли |

#### 2.1.4. Модуль дашборда (Dashboard)

| Код operation | Наименование operation | Описание |
|---------------|----------------------|----------|
| DSH-01 | Просмотр статистики | Карточки с ключевыми метриками |
| DSH-02 | Графики активности | Распределение регистраций и активности |
| DSH-03 | Лента действий | Последние операции из журнала аудита |

#### 2.1.5. Модуль журнала аудита (Audit Log)

| Код operation | Наименование operation | Описание |
|---------------|----------------------|----------|
| AUD-01 | Запись действий | Автоматическое логирование критических операций |
| AUD-02 | Просмотр журнала | Табличный вывод с фильтрацией и пагинацией |

### 2.2. Характеристика средств защиты информации

#### 2.2.1. Аутентификация

- Метод аутентификации: JWT Bearer Token
- Алгоритм подписи: HMAC-SHA256
- Хранение токена: HttpOnly Cookie (Secure, SameSite=Strict)
- Время жизни access-токена: 60 минут
- Время жизни refresh-токена: 7 дней

#### 2.2.2. Хранение паролей

- Алгоритм хеширования: BCrypt
- Автоматическая генерация соли
- Минимальные требования к паролю: 8 символов, заглавная буква, строчная буква, цифра, спецсимвол

#### 2.2.3. Защита от атак

| Механизм | Реализация |
|----------|-----------|
| Защита от CSRF | SameSite=Strict cookies + AntiForgeryToken |
| Защита от XSS | HttpOnly cookies, автоматическое экранирование вывода |
| Защита от подбора пароля | Блокировка аккаунта после неудачных попыток |
| Защита от повторного воспроизведения | Уникальный jti claim в каждом JWT |

## 4. ОПИСАНИЕ ИНТЕРФЕЙСОВ

### 4.1. Вход в систему

**URL:** `/Account/Login`

**Поля ввода:**
- Имя пользователя или Email (обязательное)
- Пароль (обязательное)
- «Запомнить меня» (флажок)

**Действия:**
- «Войти» — аутентификация и вход
- Ссылка «Забыли пароль?» — переход к сбросу пароля
- Ссылка «Регистрация» — переход к регистрации

### 4.2. Регистрация

**URL:** `/Account/Register`

**Поля ввода:**
- Имя (необязательное)
- Фамилия (необязательное)
- Имя пользователя (обязательное)
- Email (обязательное)
- Пароль (обязательное)
- Подтверждение пароля (обязательное)
- «Согласен с условиями» (флажок)

### 4.3. Dashboard

**URL:** `/Home/Dashboard`

**Элементы:**
- Карточки статистики (4 штуки)
- Таблица последних действий
- Диаграмма распределения по ролям

### 4.4. Управление пользователями

**URL:** `/Users/Index`

**Элементы:**
- Форма поиска и фильтрации
- Таблица пользователей с действиями
- Пагинация

### 4.5. Управление ролями

**URL:** `/Roles/Index`

**Элементы:**
- Таблица ролей с действиями
- Форма создания/редактирования роли
- Форма управления разрешениями (дерево чекбоксов)

### 4.6. Журнал аудита

**URL:** `/Audit/Index`

**Элементы:**
- Форма фильтрации (действие, статус, период)
- Таблица записей аудита
- Пагинация

### 5.1. Требования к среде

| Компонент | Требование |
|-----------|-----------|
| Операционная система | Windows 10+, Linux, macOS |
| .NET SDK | 10.0 |
| PostgreSQL | 18+ |
| Браузер | Chrome 90+, Firefox 88+, Edge 90+ |

### 5.2. Порядок установки

1. Установить .NET SDK 10.0
2. Установить PostgreSQL 18
3. Создать базу данных:
   ```sql
   CREATE DATABASE jwtauthdemodb;
   ```
4. Применить миграции:
   ```bash
   dotnet ef database update
   ```
5. Запустить приложение:
   ```bash
   dotnet run
   ```
6. Перейти по адресу http://localhost:5044
